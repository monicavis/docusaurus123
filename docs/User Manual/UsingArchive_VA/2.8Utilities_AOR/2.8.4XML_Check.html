2.8.4 Metadata XML Validation

You can use this utility to generate the commands that will help you to create metadata XML file for those archive files which are missing Metadata XML.

XML Script Generation​

The folders might contain some archive files with XML files and some without XML files.for those files which are missing theMetadata XML a script will be created which will the generate the XML file.

This process is specifically for windows as the script needs to be ran in same machine which has 64 bit Optim server , and this feature will be enabled only when there is a windows Convert server

Do the following to generate the Script for XML Creation:

  1. Select the Windows server on which you need to run the XMLScript Generation.

2. Click Browse. The File structure dialog box appears.

  1. Select appropriate folder/file from the mounted folder location on which you need to run the scan.

  2. Double-click the folder and click Select current folder.

  1. Click Validate. An inline message appears.
  • Metadata XML processing in the background.

Click the View Report to navigate to utility monitor.

  • If the validation is unsuccessful, an inline message appears.

Click the View icon. The Reports dialog box displays that commands that got generated for the selected archive file(s).

  1. Based on the validation status, click either Cancel or OK to navigate to the XML check page

  2. First 2 line of the report show the location of the script file in both linux and windows

  3. You should run this command from windows server to generate the xml file for the selected archive files.

note

You need to change the extension of the file from .txt to .cmd and make sure to add PSTHOME/bin to Path location in windows convert server.
