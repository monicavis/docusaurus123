2.5.1 Modifying Conversion Server

Do the following to modify the Conversion Server details:

  1. Click Manage connections on the dashboard. The Server configurations page appears.

  2. Click the Convert server tab.

  1. Click the Edit icon.

  2. Modify the below information, if necessary:

  • Optim server name:Type the server name from the optim configuration.

  • PSTHOME path:Type the corresponding PSTHOME path that points to rtsetenv.

  • Optim directory: Type the Optim directory name as it is in the optim configuration.

  • Destination Location: Type the destination location, where the generated CSV and Parquet file are stored.

  • Default qualifier: Type the default qualifier, which is a valid DB alias name with valid schema name in dbalias.schema format

    • Platform Type: one can be selected from Windows and Linux.

    • Source Location: Type the source location, where archive files exist. This will be network drive location if Platform Type is windows.

    • Destination Location: Type the destination location, where the generated CSV and Parquet file are stored. This will be network drive location if Platform Type is windows.

    • Source mount path: This filed come up only if Platform Type is windows, enter the mounted path of the Source location’s network drive.

    • Destination mount path: This filed come up only if Platform Type is windows, enter the mounted path of the destination location’s network drive.

  1. Click Update server to save the changes or Cancel to discard the changes.

  1. Click Done.

  2. Click Add server to add the conversion server details.

    • You can add multiple Convert servers by selecting Add new server and repeating the previously mentioned steps for each new server.
