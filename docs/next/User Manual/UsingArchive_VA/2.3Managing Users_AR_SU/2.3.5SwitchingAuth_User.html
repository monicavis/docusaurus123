2.3.5 Switching Authentication for a User

User can switch authentication from Basic to LDAP or vice versa, if required.

To switch the user authentication, follow these steps:

  • Login as a super user or as an administrator. The dashboard appears as per user respective role.

  • Now click on Manage users module.

  • Click Switch authentication. The Switch authentication page appears.

  • Provide the IP address in the Server IP/DNS field.

  • Provide the port number in the Port field.

  • Provide the username and password in the Privileged user DN and Password fields, respectively.

  • Select the Use TLS checkbox to connect via LDAPs, otherwise, leave it unselected.

  • Provide the DN username in the Base DN field from where the application should search for the user.

  • Provide attribute value in the Search by username attribute field based on which the username will be validated.

  • Click Test. On successful connection, the Success message dialog box appears.

  • Click Done.

  • Click Next to add a user. The Success message appears as in the below dialog box.

  • Click Done. The super user will be logged out and the Sign in page appears.

  • Provide the newly created LDAP credentials in the Username and Password fields.

  • Click Login. The dashboard applicable for super user appears.

  • Click Manage users to navigate to the Users page.

  • Click Add new. The Import user page appears.

  • Select Administrator from the Select role drop-down field.

  • Select the username attribute from the Select attribute drop-down field.

  • Provide the username in the Search by field and click Search to search LDAP users.

  • If you click on search, dialog box will appear with password field. Logged in user needs to enter his/her password to proceed

A list of LDAP users is displayed as shown below.

  • Select any user and click Import user to assign respected roles. The Import user dialog box appears.

  • Click OK. The imported LDAP user details are updated in the Users page as shown below.
