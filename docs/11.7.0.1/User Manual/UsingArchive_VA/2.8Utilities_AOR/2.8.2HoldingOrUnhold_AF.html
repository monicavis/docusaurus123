2.8.2 Holding or Un-holding Archive Files from Deletion

You can hold or un-hold the deletion of archive file(s) based on the requirement using the Delete hold tab.

Holding Archive Files from Deletion​

Do the following to hold Parquet files that are derived from an archive file from deletion:

  1. Click Utilities on the dashboard. The Utilities page appears.

  2. Click the Delete hold tab.

  1. Select an application name in which you do not want to delete the Parquet files from the Application name drop-down list.

  1. Select the archive file(s) you want to hold from deletion in the Files that are not held from deletion section.

note

You can use the chevron icon of any archive file to view the associated Parquet files.

  1. Click Hold if you want to hold only the selected files or Hold all to hold all the files. The Success message dialog box appears.

  1. Click OK. The files are moved to the Files which are held from deletion section.

Un-holding an Archive File from Deletion​

Do the following to hold an archive file from deletion:

  1. Click Utilities on the dashboard. The Utilities page appears.

  2. Click the Delete hold tab.

  1. Select an archive application name in which you want to delete archive files from the Application name drop-down list

  1. Select the file(s) you want to un-hold from deletion in the Files which are held from deletion section.

note

You can use the chevron icon of any archive file to view the associated Parquet files.

  1. Click Unhold if you want to un-hold only the selected files or Unhold all to un-hold all the files. The Success message dialog box appears.

  1. Click OK. The files are moved to the Files that are not held from deletion section.
